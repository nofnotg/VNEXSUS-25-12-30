## 📊 케이스당 평균 데이터

### 문서 특성
| 항목 | 평균값 | 범위 |
|------|--------|------|
| **페이지 수** | 15페이지 | 5-30페이지 |
| **이미지 크기** | 2-3MB | 1-5MB |
| **OCR 블록 수** | 150-300개 | 50-500개 |
| **텍스트 길이** | 3000-5000자 | 1000-8000자 |
| **입력 토큰** | 6000 tokens | 2000-10000 tokens |
| **출력 토큰** | 300 tokens | 100-500 tokens |

### 토큰 계산 기준
```
Input 토큰 = OCR 텍스트 (6000자 × 0.5 토큰/자) + 프롬프트 (500 토큰)
            ≈ 3000 + 500 = 3500 tokens (실제 6000으로 여유 있게 계산)

Output 토큰 = JSON 응답 {"dates": ["2024-05-01", ...]}
            ≈ 300 tokens
```

### 가격 책정 원칙

#### 1. 원가 기반 (Cost-Plus)
```
구독 가격 = API 원가 × 마진 배수
```

**일반적인 SaaS 마진:**
- Freemium 모델: 5-10배
- B2B 엔터프라이즈: 10-20배
- AI 서비스: 3-5배 (API 원가 비중 높음)

#### 2. 가치 기반 (Value-Based)
```
구독 가격 = 고객이 얻는 가치 (시간 절약, 정확도 향상)
```
