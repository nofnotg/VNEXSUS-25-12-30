### 5.2 개선 방안

#### 🚀 **Phase 1: 긴급 (1주 이내) - 날짜 검증 로직 추가**

**목표**: 잘못된 날짜 필터링 및 경고

**구현**:

```typescript
// src/utils/dateValidator.ts
export function validateDate(dateStr: string): {
  isValid: boolean;
  normalized?: string;
  error?: string;
} {
  const match = dateStr.match(/^(\d{4})-(\d{1,2})-(\d{1,2})$/);
  if (!match) return { isValid: false, error: 'Invalid format' };

  const [_, year, month, day] = match;
  const y = parseInt(year);
  const m = parseInt(month);
  const d = parseInt(day);

  // 연도 범위 검증 (1900-2100)
  if (y < 1900 || y > 2100) {
    return { isValid: false, error: `Invalid year: ${y}` };
  }

  // 월 범위 검증 (1-12)
  if (m < 1 || m > 12) {
    return { isValid: false, error: `Invalid month: ${m}` };
  }

  // 일 범위 검증
  const daysInMonth = new Date(y, m, 0).getDate();
  if (d < 1 || d > daysInMonth) {
    return { isValid: false, error: `Invalid day: ${d} (max ${daysInMonth} for ${y}-${m})` };
  }

  return {
    isValid: true,
    normalized: `${y}-${String(m).padStart(2, '0')}-${String(d).padStart(2, '0')}`
  };
}
```

**적용 위치**:
- `src/modules/medical-events/service/pipelineAdapter.ts` - 이벤트 생성 시
- `backend/eval/report_subset_validator.js` - 검증 시

**예상 효과**:
- 잘못된 날짜 100% 필터링
- 검증 통과율 71.4% → 95%+ 향상

```typescript
// src/services/dateCorrector.ts
import OpenAI from 'openai';

export async function correctDates(
  text: string,
  suspiciousDates: string[]
): Promise<Map<string, string>> {
  const client = new OpenAI({ apiKey: process.env.OPENAI_API_KEY });

  const prompt = `다음 의료 문서에서 의심스러운 날짜를 교정하세요:

의심스러운 날짜: ${suspiciousDates.join(', ')}

문서 일부:
${text.slice(0, 1000)}

각 날짜에 대해 올바른 날짜를 추론하세요. JSON 형식으로 응답:
{ "원본": "교정된_날짜" }

예: { "2049-03-24": "2024-03-24", "2025-03-45": "2025-03-15" }`;

  const response = await client.chat.completions.create({
    model: 'gpt-4o-mini',
    messages: [
      { role: 'system', content: '당신은 의료 문서 날짜 교정 전문가입니다.' },
      { role: 'user', content: prompt }
    ],
    temperature: 0.1
  });

  const result = JSON.parse(response.choices[0].message.content || '{}');
  return new Map(Object.entries(result));
}
```

**파이프라인 통합**:
1. OCR 텍스트 추출
2. 날짜 추출
3. **날짜 유효성 검증** → 의심스러운 날짜 목록 생성
4. **LLM 교정** → 교정된 날짜 맵 생성
5. 이벤트 생성 (교정된 날짜 사용)

**예상 비용**:
- 케이스당 LLM 호출: 1회
- 토큰 수: ~500 tokens
- 비용 (gpt-4o-mini): $0.00015/케이스
- 7개 케이스: $0.001

**예상 효과**:
- 검증 통과율 95% → 99%+ 향상
- OCR 오류 자동 교정
