# 02. Target Output Spec (심사기준 우선 VNEXSUS 출력 명세)

## A. VNEXSUS(전체집합) 출력 목표
- “모든 의료이벤트”를 제공하되, **심사기준 우선 정렬**로 조사자가 빠르게 쟁점을 잡도록 한다.
- report는 subset이므로, VNEXSUS 안에 report에 등장한 모든 핵심 사실(날짜/코드/병원/검사/수술)이 **정확히 포함**되어야 한다.

## B. 출력 레이어(3단)
1) **Core Events (심사 핵심 이벤트)**
   - 가입 전 3개월/5년 구간의 중대 이벤트(수술/입원/중대검사/암/심혈관 등)
   - 고지의무 질문에 직접 연동되는 이벤트(질환군·치료/검사 트리거)
2) **Episode Summary (사건/에피소드 단위 압축)**
   - 동일 병원·동일 질환 클러스터의 반복 진료를 기간/횟수로 묶고,
   - 그 중 “가장 무거운 이벤트”(입원/수술/중대검사/의사소견)를 표시
3) **Full Timeline (전체 날짜 이벤트)**
   - 어떤 이벤트도 자유요약에서 새로 만들어내지 않음
   - 각 항목은 최소한 `date, hospital, diagnosisName(가능 시 code), eventType, shortFact, sourceSpan`을 갖는다.

## C. 고지의무(심사기준) 중심 섹션
VNEXSUS 출력 상단에 아래 섹션을 추가한다.

### 1) 가입일 기준 중요 구간 요약
- 가입 전 3개월 핵심 이벤트
- 가입 전 5년 핵심 이벤트
- 가입 후 청구 관련 이벤트(가능하면 별도 표기)

### 2) 고지의무 질문 맵(Question Map)
- 예: “최근 5년 이내 입원/수술/추가검사 여부”
- 예: “암/종양 진단/검사/치료 여부”
- 예: “심혈관(협심증/심근경색/관상동맥) 진단/검사 여부”
각 질문별로:
- 매칭 이벤트 목록(이벤트 ID, 날짜, 요약, 근거)
- 판단 근거(룰 이름/조건)

### 3) 근거 추적(Evidence Trace)
- 모든 핵심 이벤트는 원문에서 근거구간을 갖는다:
  - `sourceSpan: { start, end, textPreview }`
- 사용자 UI에서 클릭 시 원문 하이라이트 가능하도록 설계(추후).

## D. 절대 금지
- 원문에 없는 진단명/코드/검사 결과를 생성하지 않는다.
- 날짜가 불명확하면 불명확(미상) 상태로 유지하고 “추정”을 하지 않는다(정밀도 우선).
